[1:00 - ARCHITECTURE]
(show assets/architecture_diagram.png)
Let us walk through the architecture of the system.
Every parenting query starts at the security checkpoint node, which de-identifies contact information, detects prompt injections, and audits severity.
If cleared, the orchestrator agent takes over to classify the intent and delegate the query to the best specialist sub-agent.
The developmental coach, mental health advocate, and disability support agent then call domain-specific tools via local Model Context Protocol or MCP connections to retrieve facts.
Before the parent sees any advice involving schedules or discipline rules, the flow passes through a human approval gate, prompting the parent to review and authorize the recommendations.
Finally, the output is generated and logged securely.
